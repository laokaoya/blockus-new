# Blockus 待提升项总结

> 基于当前版本与可发布版本的差距评估

---

## 一、质量与稳定性（P0）

| 项目 | 现状 | 目标 | 状态 |
|------|------|------|------|
| ErrorBoundary | 无，组件崩溃会白屏 | 添加全局 ErrorBoundary，展示友好错误页 | ✅ 已实现 |
| 自动化测试 | 无 `*.test.*` 文件 | 核心逻辑（放置规则、AI、道具效果）补充单元测试 | ✅ gameEngine、creativeModeEngine 已补 |
| 代码分割 | 全量打包 | 路由级 lazy loading，降低首屏体积 | ✅ 已实现 |
| Socket 重连 | 有重连逻辑，用户无感知 | 断线时 Toast 提示，重连成功提示 | ✅ ConnectionStatus 已实现 |

---

## 二、用户体验与引导（P1）

| 项目 | 现状 | 目标 |
|------|------|------|
| 新手引导 | 无 | 首次进入简短规则/操作引导（可跳过） |
| 空状态 | 部分列表无空态 | 房间列表、对局记录等增加空状态 |
| 加载态 | 部分异步无 loading | 进入房间、开始游戏等增加 loading/骨架屏 |
| 操作撤销 | PRD 要求「确认前可撤销」 | 落子确认前支持撤销 |

## 实施优先级建议

1. **P0 质量与稳定性**：ErrorBoundary、测试、代码分割、重连 UX
2. **P1 体验与安全**：引导、空状态、加载态、输入校验、错误监控
3. **P2 功能与优化**：匹配、等级、无障碍、PWA、国际化
